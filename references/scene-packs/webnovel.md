# 网络文学场景包 · 本地内置规范（降级用）

> **体裁**：webnovel · **版本**：1.0
> **说明**：本文件为四级降级路径第 2 级（内置规范）兜底内容。
> 网络不可用、远端 schema 版本不匹配、或处于本地模式时自动使用。
> **适用体裁**：网络小说（webnovel）、长篇连载（serial）
> **支持平台**：番茄小说、起点中文网、晋江文学城

---

## S0 初始化收集要素

写网文前，需收集以下信息（标注★为必须）：

1. ★ 作品类型（原创 / 同人 / 衍生）
2. ★ 目标平台（番茄小说 / 起点中文网 / 晋江文学城 / 其他）
3. ★ 小说题材与流派（玄幻 / 都市 / 言情 / 科幻 / 历史 / 悬疑 / 轻小说 等）
4. ★ 预计篇幅（短篇 10-30 万字 / 中篇 30-80 万字 / 长篇 80-300 万字 / 超长篇 300 万字以上）
5. ★ 金手指 / 核心设定（穿越 / 重生 / 系统 / 异能 / 修仙 等，一句话概括）
6. 已有存稿情况
7. 更新频率预期（日更 / 周更 / 不定）
8. 签约意向（是否计划签约平台的激励 / 分成计划）

## S2 大纲结构（网文通用）

### 三幕式 + 打点结构

```
开篇（第 1-10 章）— 黄金三章必杀
  ├── 第一章 钩子（悬念 / 冲突 / 穿越 / 反差，第 1 段即需抓住读者）
  ├── 第二章 金手指登场（系统 / 能力 / 身份反转）
  ├── 第三章 小高潮（第一个冲突解决或升级前奏）
  └── 4-10 章 世界观展开 + 角色图鉴穿插

中盘（第 XX 章 ~ 80% 篇幅）
  ├── 一个大主线贯穿 + 单元副本交替
  ├── 每 10-20 章一个小高潮（升级 / 战boss / 身份揭秘）
  ├── 每章结尾留钩子（悬念 / 危机 / 新线索）
  └── 避免超过 5 章无爽点

高潮 → 收尾
  ├── 最终冲突 / 大决战
  ├── 埋线回收（所有伏笔至少 80% 收束）
  └── 结局（HE / BE / 开放式，据平台受众定）
```

### 黄金三章规范（P0 强制）

| 要素 | 第 1 章 | 第 2 章 | 第 3 章 |
|------|---------|---------|---------|
| 主角出场 | ✅ 必须 | — | — |
| 核心冲突 | ✅ 展现 | ✅ 升级 | ✅ 第一次解决 |
| 金手指 | 埋线 | ✅ 清晰展示 | — |
| 悬念/钩子 | ✅ 开头即有 | ✅ 章尾 | ✅ 章尾 |
| 读者代入感 | ✅ 建立 | ✅ 强化 | ✅ 巩固 |
| 字数 | ≥ 2000 | ≥ 2000 | ≥ 2000 |

## 各平台交付规范

### 番茄小说（Fanqie Novel / 番茄作家助手）

**格式要求：**
- 每章 2000-3000 字为最佳区间（平台推荐）
- 纯文本格式，禁止富文本（加粗、斜体、彩色文字）
- 段落不宜超过 5 行
- 对话多用短段，一段一句对话为佳
- 禁止首行缩进，段间空一行
- 章节标题格式：`第XX章 XXXXX`

**内容风格：**
- 直白通俗，初中以上阅读水平
- 节奏快：每章至少一个小钩子
- 单女主 / 无女主更受欢迎（男频）
- 标签化：系统、签到、无敌流、赘婿流（男频）| 甜宠、重生、宫斗、穿书（女频）
- 禁止敏感内容（时政、真名负面、色情细则）
- 前后文衔接紧密，每章结尾必须留钩子引导下一章

**元数据：**
- 作品简介：100-200 字，突出核心卖点
- 分类标签：精确到子类（如 都市 > 生活 > 系统）
- 封面要求：1:1 比例，不含文字 / 二维码

**更新与分发：**
- 日更 4000-6000 字（2-3 章）为推荐频率
- 章节自动同步至番茄小说 App
- 签约后可进入分成推广池

---

### 起点中文网（Qidian / 阅文集团）

**格式要求：**
- 每章 2000-4000 字
- VIP 章节每章不少于 2000 字
- 章节标题格式：`第XX章 XXXXX` (VIP 章节可直接用数字序号)
- 正文前空两格（全角空格）
- 段间空一行

**内容标准：**
- 开篇即入题，避免大段背景说明
- 首章 3000 字以上为宜
- 前 3 万字内完成设定展开
- 推荐票 / 月票 / 打赏节点设计（起点的互动机制）
- 付费章节节奏更紧凑，每章节末尾必须有继续阅读的驱动力
- 上架（VIP）章节需保持日更 4000+ 字

**元数据与运营：**
- 笔名、简介（200 字内）、标签（最多 5 个）
- 分类：五大类 + 子类（玄幻·奇幻·武侠·仙侠·都市·现实·军事·历史·游戏·体育·科幻·悬疑·轻小说）
- 周推荐票榜、月票榜、新人榜单竞争激烈

**独家规范：**
- 全勤奖条件：日更 4000 字 + 月更 12 万字
- 大纲需提前准备前 20 章 + 后 30 章规划（编辑签约评估用）
- 签约后可申请 VIP 上架，获得分成收入
- 推荐机制：编辑推 + 数据推（追读率是核心指标）

---

### 晋江文学城（JJWXC / 晋江原创网）

**格式要求：**
- 每章 3000-5000 字（范晋江圈推荐区间）
- 标题格式：`第XX章 XXXXX`或直接使用章节名
- 正文内引号使用「」（直角引号，晋江惯例）
- 内容简介：100-500 字
- 文案（文前引流文案）有独立区域

**内容特色：**
- 女性向为主：言情 / 耽美 / 百合 / 无CP
- 细分标签系统极丰富（古穿今、重生复仇、甜宠、追妻火葬场、白月光、替身…）
- 情感描写细腻，心理活动丰富
- 节奏比番茄慢但读者黏性更高
- 评论区互动重要（评分机制影响曝光）

**分级与审核：**
- 内容分级：全年龄 / 限制级（需标注）
- 清水文与肉文的分界，擦边内容需按平台最新规则处理
- 敏感题材：涉政、涉军、侵权、真名真实事件需回避
- 文案和内容不能出现具体金额 / 第三方平台引流

**签约与收入：**
- 积分制决定榜单排序
- 签约后可进入 VIP 章节收费
- 千字 3-5 分（普通作者），大神千字可达 1 角
- 月榜、季榜、年度盘点

### 多平台通用规范

| 项目 | 规范 |
|------|------|
| 每章字数下限 | ≥ 2000 字（网文通用标准） |
| 每章字数上限 | 建议不超过 5000 字（移动端阅读体验） |
| 段落长度 | 不超过 5 行，对话尽可能 1-2 行一段 |
| 章末钩子 | 每章结尾必须留悬念 / 反转 / 新问题 |
| 章节节奏 | 每 10 章必须有一个小高潮，每 30 章一个中高潮 |
| 更新频率 | 日更 4000-6000 字（读者习惯期待） |
| 敏感内容 | 不涉时政真名、不涉黄色细则、不涉侵权 |
| 存稿要求 | 至少 10 章存稿再开始连载（应对断更压力） |
| 排版 | 纯文本，标点统一用全角，数字用半角 |
| 标签 | 精确到子类，不使用语义模糊的标签（如「好看」） |

## 检索与素材策略

- 搜索同类热门作品的标签组合、文案标题
- 搜索目标平台最新的榜单前 50 名，分析高频标签
- 搜索「金手指」设定库、梗百科（如 起点金手指数据库）
- 搜索读者讨论区反馈，定位当前流行趋势
- 搜索各平台的「避雷」帖，避免踩坑
